## Getting Started

This is a [Next.js](https://nextjs.org/) project bootstrapped with `pgpm init constructive-app`.

**Features:**
- Next.js 16.1.1 with App Router
- React 19.2.3
- TypeScript
- Tailwind CSS v4
- ESLint with Next.js config
- Google Fonts (Geist Sans & Geist Mono)

### Prerequisites

- Node.js 20+
- pnpm (recommended) or npm

### Quick Start

```sh
# Install dependencies
pnpm install

# Run the development server
pnpm dev

# Open http://localhost:3000 in your browser
```

### Available Scripts

- `pnpm dev` - Start development server
- `pnpm build` - Build for production
- `pnpm start` - Start production server
- `pnpm lint` - Run ESLint
- `pnpm type-check` - Run TypeScript type checking

### GraphQL SDK

This template includes an auto-generated GraphQL SDK for Constructive in `src/lib/graphql/constructive/`. The SDK provides:

- **421 mutation hooks** - For creating, updating, and deleting entities
- **264 query hooks** - For fetching data from the GraphQL API
- **Type-safe** - Full TypeScript support with generated types
- **React hooks** - Ready-to-use React hooks for queries and mutations

#### Configuration

Configure the GraphQL client before use:

```ts
import { configure } from '@/lib/graphql/constructive';

configure({
  endpoint: 'https://api.example.com/graphql',
  headers: {
    Authorization: 'Bearer <token>',
  },
});
```

#### Usage Example

```tsx
import { useUsersQuery, useCreateUserMutation } from '@/lib/graphql/constructive';

function UsersPage() {
  const { data, isLoading } = useUsersQuery({ first: 10 });
  const { mutate: createUser } = useCreateUserMutation();

  const handleCreate = () => {
    createUser({
      input: {
        username: 'newuser',
        email: 'user@example.com',
      },
    });
  };

  // ...
}
```

**Note:** The GraphQL SDK is auto-generated by `@constructive-io/graphql-codegen`. To regenerate it, run:

```sh
pnpm codegen
```
